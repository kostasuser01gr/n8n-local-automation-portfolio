# n8n Application Final Cover Note

Repository: https://github.com/kostasuser01gr/n8n-local-automation-portfolio

I built a verified local-first n8n automation portfolio to demonstrate workflow engineering, PostgreSQL persistence, self-hosted operations, debugging, release packaging, and documentation quality.

The repository contains three executed workflows: Job Application Intelligence, Fleet Operations, and Local AI Knowledge. Each workflow includes sanitized exports, sample data, screenshots, case-study documentation, and persistence evidence.

Verified counts are small by design: 2 job application rows, 3 fleet vehicle rows, and 3 knowledge document rows, with one run record for each workflow path. These counts demonstrate execution and persistence, not scale.

The work is relevant to n8n because it shows the habits needed for workflow platform work: reproduce issues, inspect logs and data, keep credentials out of release artifacts, document setup clearly, and package examples that other builders can review.
